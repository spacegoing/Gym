# Plan: YAML Schema + docs/data for NeMo Gym Environments

**Constraint:** Create a new, separate script. Do not modify `scripts/update_resource_servers.py`.

---

## 1. Goals

- **Single source of truth**: New script produces structured data for docs; existing script continues to update README only.
- **Zero impact on existing flow**: `update_resource_servers.py` and its pre-commit hook remain unchanged.
- **Better docs**: Structured data in `docs/data/` for Sphinx and future tooling.
- **Reuse**: New script imports from existing script to avoid logic duplication.

## 5. Pre-commit and CI

### 5.1 Pre-commit options

**Option A: Separate hook (recommended)**  
Add a second hook that runs only `generate_environments_yaml.py`:

```yaml
- id: generate-environments-yaml
  name: Generate docs/data/environments.yaml
  entry: python scripts/generate_environments_yaml.py
  language: system
  files: ^resources_servers/
```

Runs when `resources_servers/` changes; keeps README and YAML in sync.

**Option B: Manual / CI only**  
No pre-commit; run script manually or in CI before docs build.

### 5.2 CI

- Run `python scripts/generate_environments_yaml.py`
- Fail if `docs/data/environments.yaml` has uncommitted changes
- Or run the pre-commit hook in CI
